0. RESUMEN EJECUTIVO
========================================

OBJETIVO:
Crear un juego mobile simple, publicable y escalable, enfocado en retencion, progresion constante y monetizacion mediante anuncios recompensados, compras opcionales y eliminacion de anuncios.

TIPO DE JUEGO RECOMENDADO:
- Idle Clicker / Incremental 2D.
- Sesiones cortas de 30 segundos a 5 minutos.
- Progreso offline.
- Recompensas diarias.
- Eventos temporales sencillos.

PRIORIDAD DEL PROYECTO:
1. Lanzar un MVP funcional.
2. Medir retencion y comportamiento real.
3. Ajustar economia y monetizacion.
4. Agregar contenido visual, eventos y progresion.

PRINCIPIO CLAVE:
No intentar crear el juego mas grande desde el inicio. Crear un producto pequeno, entendible, pulido y medible.

========================================
1. CONCEPTO DEL JUEGO
========================================

IDEA BASE:
Un juego incremental donde el jugador toca un elemento principal para generar monedas, compra mejoras, desbloquea generadores automaticos y avanza por etapas visuales cada vez mas llamativas.

CONCEPTO TEMATICO RECOMENDADO:
"Fabrica de Cristales Cosmica"

El jugador empieza tocando un pequeno cristal en una mesa de laboratorio. Con cada mejora, la fabrica evoluciona: herramientas, maquinas, robots, portales, planetas y tecnologia cosmica.

POR QUE ESTA TEMATICA FUNCIONA:
- Permite progresion visual clara.
- Facilita imagenes generadas por IA.
- Funciona bien en 2D.
- No depende de personajes complejos.
- Puede tener colores brillantes y efectos satisfactorios.
- Escala bien con upgrades: cristal, maquina, robot, fabrica, planeta, galaxia.

OTRAS TEMATICAS POSIBLES:
- Tienda de pociones magicas.
- Granja de slimes.
- Restaurante espacial.
- Mina de gemas.
- Fabrica de dulces.
- Laboratorio de criaturas.

RECOMENDACION:
Elegir una tematica visualmente fuerte y facil de producir. Para un primer juego, "Fabrica de Cristales Cosmica" es buena porque requiere pocos personajes y permite muchos objetos/maquinas generados por IA.

5. CORE LOOP
========================================

LOOP PRINCIPAL:
1. El jugador toca el cristal central.
2. Gana monedas.
3. Compra mejoras.
4. Aumenta monedas por toque o monedas por segundo.
5. Desbloquea nuevas maquinas/zonas.
6. Recibe recompensas offline al volver.
7. Mira anuncios recompensados si quiere acelerar progreso.
8. Repite con metas mas grandes.

LOOP DE 30 SEGUNDOS:
- Tocar.
- Comprar una mejora barata.
- Ver efecto visual inmediato.
- Ver que el contador sube mas rapido.

LOOP DE 5 MINUTOS:
- Ahorrar para una mejora importante.
- Desbloquear una nueva maquina.
- Reclamar recompensa diaria o anuncio recompensado.

LOOP DE 24 HORAS:
- Entrar por recompensa diaria.
- Reclamar progreso offline.
- Revisar evento o mision diaria.

========================================
6. SISTEMAS PRINCIPALES
========================================

[1] SISTEMA DE MONEDA
- Moneda principal: Cristales.
- Variable base: coins.
- Mostrar con formato compacto: 1K, 1M, 1B.
- Usar double o decimal si los numeros crecen mucho.

[2] SISTEMA DE TOQUE
- Cada toque genera monedas.
- Puede tener multiplicadores temporales.
- Debe sentirse satisfactorio: particulas, sonido, vibracion ligera.

[3] SISTEMA DE MEJORAS
- Mejoras de click: aumentan monedas por toque.
- Mejoras idle: aumentan monedas por segundo.
- Mejoras visuales: cambian el escenario o maquina.
- Mejoras premium opcionales: aceleran, pero no bloquean el progreso basico.

[4] SISTEMA IDLE
- Genera monedas automaticamente.
- Se calcula cada segundo.
- Mostrar monedas por segundo en la UI.

[5] PROGRESO OFFLINE
- Guardar hora de salida.
- Al volver, calcular tiempo transcurrido.
- Dar recompensa offline con limite recomendado de 8 a 12 horas.
- Opcion: duplicar recompensa viendo anuncio.

[6] SISTEMA DE GUARDADO
- MVP: PlayerPrefs para prototipo.
- Version publicable: JSON local con versionado.
- Si hay compras reales, validar recibos con Google Play cuando sea posible.

[7] SISTEMA DE PRESTIGIO
- Se desbloquea cuando el jugador alcanza una meta alta.
- Reinicia progreso principal.
- Da moneda especial permanente.
- Aumenta produccion global.

NOMBRE SUGERIDO:
- Polvo Estelar.

[8] SISTEMA DE MISIONES
- Misiones diarias:
    - Toca 500 veces.
    - Compra 10 mejoras.
    - Reclama 3 recompensas.
    - Gana X cristales.

- Recompensas:
    - Monedas.
    - Multiplicador temporal.
    - Cofre comun.

[9] SISTEMA DE COFRES
- Cofre comun: monedas, boost pequeno.
- Cofre raro: multiplicador, moneda de prestigio pequena.
- Cofre evento: cosmeticos o fondos.

IMPORTANTE:
Si se usan mecanicas tipo cofre aleatorio, revisar politicas de Google Play sobre probabilidades y compras.

========================================
7. ECONOMIA DEL JUEGO
========================================

VARIABLES PRINCIPALES:
- coins: moneda actual.
- totalCoinsEarned: total historico.
- clickPower: monedas por toque.
- coinsPerSecond: produccion automatica.
- prestigeCurrency: moneda permanente.
- multiplier: multiplicador global.

FORMULA SIMPLE DE COSTO DE UPGRADE:
costo = costoBase * pow(crecimiento, nivel)

EJEMPLO:
- costoBase = 10
- crecimiento = 1.15
- nivel = nivel actual del upgrade

FORMULA DE PRODUCCION:
produccion = produccionBase * nivel * multiplicadorGlobal

BALANCE INICIAL RECOMENDADO:
- Primer upgrade disponible en menos de 20 segundos.
- Segundo upgrade en menos de 1 minuto.
- Primer generador automatico antes de 2 minutos.
- Primera recompensa offline visible en la primera vuelta del jugador.
- Primer prestigio entre 45 y 90 minutos acumulados, no necesariamente en una sola sesion.

CURVA DE PROGRESION:
- Inicio rapido.
- Medio con decisiones: click vs idle.
- Tarde con prestigio y multiplicadores.

RIESGO:
Si el progreso se vuelve demasiado lento, el jugador abandona. Si es demasiado rapido, se queda sin objetivos. El balance se debe ajustar con datos reales.

========================================
8. MONETIZACION RESPONSABLE
========================================

OBJETIVO:
Generar ingresos sin destruir la experiencia. La monetizacion debe sentirse opcional, clara y justa.

[ADS]

TIPOS RECOMENDADOS:
- Rewarded Ads: prioridad maxima.
- Interstitial Ads: usar con mucho cuidado.
- Banner Ads: opcional, no recomendado si ensucia la UI.

REWARDED ADS:
- Duplicar recompensa offline.
- Dar monedas extra.
- Activar multiplicador x2 por 5 minutos.
- Abrir cofre adicional.

INTERSTITIAL ADS:
- No mostrar en los primeros 3 a 5 minutos.
- No mostrar despues de cada toque o accion frecuente.
- Mostrar solo al cambiar de pantalla o tras completar una meta.
- Frecuencia recomendada inicial: maximo 1 cada 3 a 5 minutos.

[IAP]

PRODUCTOS RECOMENDADOS:
- Pack pequeno de cristales.
- Pack mediano de cristales.
- Pack grande de cristales.
- Remove Ads.
- Starter Pack.
- Multiplicador permanente ligero.

PRECIOS SUGERIDOS INICIALES:
- Pack pequeno: USD 0.99
- Pack mediano: USD 2.99
- Pack grande: USD 4.99 o 9.99
- Remove Ads: USD 2.99
- Starter Pack: USD 1.99

RECOMENDACION:
El producto mas importante suele ser Remove Ads. Debe eliminar interstitials y banners, pero puede mantener rewarded ads si el jugador elige verlos voluntariamente.

EVITAR:
- Pagar para desbloquear funciones basicas.
- Interrumpir constantemente con anuncios.
- Hacer que el juego sea insoportable sin pagar.
- Ofertas confusas.

GameManager.cs:

using UnityEngine;

public class GameManager : MonoBehaviour
{
    public static GameManager Instance { get; private set; }

    public double Coins { get; private set; }
    public double ClickPower { get; private set; } = 1;
    public double CoinsPerSecond { get; private set; }

    private void Awake()
    {
        if (Instance != null && Instance != this)
        {
            Destroy(gameObject);
            return;
        }

        Instance = this;
        DontDestroyOnLoad(gameObject);
        LoadGame();
    }

    private void Update()
    {
        AddCoins(CoinsPerSecond * Time.deltaTime, false);
    }

    public void ClickMainObject()
    {
        AddCoins(ClickPower, true);
    }

    public void AddCoins(double amount, bool saveImmediately)
    {
        Coins += amount;

        if (saveImmediately)
        {
            SaveGame();
        }
    }

    public bool SpendCoins(double amount)
    {
        if (Coins < amount)
        {
            return false;
        }

        Coins -= amount;
        SaveGame();
        return true;
    }

    public void IncreaseClickPower(double amount)
    {
        ClickPower += amount;
        SaveGame();
    }

    public void IncreaseCoinsPerSecond(double amount)
    {
        CoinsPerSecond += amount;
        SaveGame();
    }

    private void SaveGame()
    {
        PlayerPrefs.SetString("coins", Coins.ToString());
        PlayerPrefs.SetString("clickPower", ClickPower.ToString());
        PlayerPrefs.SetString("coinsPerSecond", CoinsPerSecond.ToString());
        PlayerPrefs.Save();
    }

    private void LoadGame()
    {
        Coins = double.Parse(PlayerPrefs.GetString("coins", "0"));
        ClickPower = double.Parse(PlayerPrefs.GetString("clickPower", "1"));
        CoinsPerSecond = double.Parse(PlayerPrefs.GetString("coinsPerSecond", "0"));
    }
}

UpgradeSystem.cs:

using System;
using UnityEngine;

public class UpgradeSystem : MonoBehaviour
{
    public double GetCost(UpgradeData upgrade, int currentLevel)
    {
        return upgrade.baseCost * Math.Pow(upgrade.costGrowth, currentLevel);
    }

    public bool BuyUpgrade(UpgradeData upgrade, int currentLevel)
    {
        double cost = GetCost(upgrade, currentLevel);

        if (!GameManager.Instance.SpendCoins(cost))
        {
            return false;
        }

        if (upgrade.clickPowerBonus > 0)
        {
            GameManager.Instance.IncreaseClickPower(upgrade.clickPowerBonus);
        }

        if (upgrade.coinsPerSecondBonus > 0)
        {
            GameManager.Instance.IncreaseCoinsPerSecond(upgrade.coinsPerSecondBonus);
        }

        return true;
    }
}

NumberFormatter.cs:

public static class NumberFormatter
{
    private static readonly string[] Suffixes = { "", "K", "M", "B", "T", "Qa", "Qi" };

    public static string Format(double value)
    {
        int suffixIndex = 0;

        while (value >= 1000 && suffixIndex < Suffixes.Length - 1)
        {
            value /= 1000;
            suffixIndex++;
        }

        return value.ToString("0.##") + Suffixes[suffixIndex];
    }
}

========================================
23. DEFINICION DE MVP
========================================

EL MVP DEBE TENER:
- Una escena jugable.
- Monedas.
- Toque principal.
- 10 upgrades.
- Monedas por segundo.
- Guardado local.
- Progreso offline.
- Recompensa diaria basica.
- Un rewarded ad funcional.
- UI presentable.
- Icono temporal.

EL MVP NO NECESITA TODAVIA:
- Muchos mundos.
- Personajes animados.
- Login.
- Backend complejo.
- Eventos avanzados.
- Cofres premium.
- Multiplayer.

CRITERIO DE EXITO DEL MVP:
Una persona externa puede instalarlo, entenderlo sin explicacion y jugar al menos 5 minutos sin trabarse.

- Sea facil de actualizar con nuevos upgrades, eventos y assets.

ENFOQUE CORRECTO:
Primero jugabilidad funcional. Luego arte. Luego monetizacion. Luego optimizacion con datos.
